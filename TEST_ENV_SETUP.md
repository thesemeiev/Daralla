# Настройка тестовой среды

## Структура

- **Ветка**: `test` - тестовая ветка
- **Сервер**: Тестовый сервер (второй сервер)
- **Workflow**: `.github/workflows/deploy-server2.yml` (теперь для тестовой среды)

## Шаг 1: Создайте тестовую ветку

```bash
# Создайте тестовую ветку
git checkout -b test

# Или если ветка уже существует
git checkout test
```

## Шаг 2: Добавьте секреты в GitHub

Перейдите: `Settings` → `Secrets and variables` → `Actions`

Добавьте секреты для тестового сервера:

- `TEST_SERVER_HOST` - IP адрес или домен тестового сервера
- `TEST_SERVER_USER` - имя пользователя для SSH (обычно `root`)
- `TEST_SERVER_PASSWORD` - пароль для SSH
- `TEST_SERVER_PORT` - порт SSH (опционально, по умолчанию 22)

**Важно:** Используйте префикс `TEST_` чтобы отличать от продакшн секретов.

## Шаг 3: Настройте тестовую среду

### На тестовом сервере:

1. **Создайте отдельную папку для тестовой среды** (опционально):
```bash
sudo mkdir -p /root/Daralla-test
```

2. **Или используйте ту же папку** - workflow автоматически сделает бэкап перед обновлением.

### В workflow (уже настроено):

- Деплой происходит при пуше в ветку `test`
- Используются секреты с префиксом `TEST_`
- Все данные автоматически бэкапятся перед обновлением

## Шаг 4: Запушьте код в тестовую ветку

```bash
# Убедитесь, что вы в тестовой ветке
git checkout test

# Добавьте изменения
git add .

# Закоммитьте
git commit -m "Test: update for testing"

# Запушьте
git push origin test
```

## Шаг 5: Проверьте деплой

1. Перейдите в `Actions` в GitHub
2. Выберите workflow "Deploy Daralla Bot to Test Server"
3. Проверьте статус деплоя

## Рабочий процесс

### Разработка:
1. Работаете в ветке `test`
2. Делаете изменения
3. Пушите в `test` → автоматический деплой на тестовый сервер
4. Тестируете на тестовом сервере

### Продакшн:
1. После тестирования мержите `test` → `main`
2. Пуш в `main` → автоматический деплой на продакшн сервер (первый сервер)

## Отличия тестовой среды

Тестовая среда полностью идентична продакшн, но:
- Использует отдельный сервер
- Использует отдельную ветку
- Можно безопасно тестировать без влияния на продакшн

## Ручной запуск деплоя

1. Перейдите в `Actions`
2. Выберите "Deploy Daralla Bot to Test Server"
3. Нажмите "Run workflow"
4. Выберите ветку `test`
5. Нажмите "Run workflow"

## Проверка после деплоя

```bash
# На тестовом сервере проверьте:
docker-compose ps
docker-compose logs telegram-bot
curl http://localhost:5000/sub/test-token
```

## Важные моменты

1. **Базы данных**: Тестовая среда имеет свои базы данных, отдельные от продакшн
2. **.env файл**: На тестовом сервере должен быть свой `.env` файл
3. **Порты**: Убедитесь, что порт 5000 свободен на тестовом сервере
4. **Домен**: Если используете домен для тестов, настройте отдельный поддомен (например, `test.ghosttunnel.space`)
