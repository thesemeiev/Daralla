# Наименование серверов

## Два типа имен:

### 1. `"name"` - Уникальный идентификатор (обязательно)
- Используется в базе данных для идентификации сервера
- Используется в коде для поиска и работы с сервером
- **Должен быть уникальным** для каждого сервера
- Рекомендуемый формат: `LOCATION-NUMBER` (например, `Latvia-1`, `Latvia-2`)

### 2. `"display_name"` - Красивое название (опционально)
- Используется только для отображения в VPN клиенте
- Может содержать эмодзи, пробелы, любые символы
- Если не указано, используется `"name"` как fallback

## Примеры:

### Правильная конфигурация:

```python
SERVERS_BY_LOCATION = {
    "Latvia": [
        {
            "name": "Latvia-1",  # Уникальный ID (обязательно, уникальный!)
            "display_name": "🇱🇻 🌐 Latvia Premium - 1",  # Красивое название (опционально)
            ...
        },
        {
            "name": "Latvia-2",  # Другой уникальный ID
            "display_name": "🇱🇻 ⚡ Latvia Speed - 2",  # Другое красивое название
            ...
        },
    ],
}
```

### Если не указать `display_name`:

```python
{
    "name": "Latvia-1",  # Будет использовано и как ID, и как название в VPN клиенте
    # display_name не указан - используется "Latvia-1"
    ...
}
```

## Важно:

- ✅ `"name"` должен быть уникальным для каждого сервера
- ✅ `"name"` используется в базе данных - не меняйте его после создания подписок!
- ✅ `"display_name"` можно менять в любое время - это только для отображения
- ✅ Если у вас несколько серверов, используйте разные `"name"`: `Latvia-1`, `Latvia-2`, `Finland-1` и т.д.

## Как это работает:

1. В базе данных сохраняется `"name"` (например, `"Latvia-1"`)
2. При генерации VLESS ссылок используется `"display_name"` (если указано) или `"name"` (если не указано)
3. В VPN клиенте отображается красивое название из `"display_name"`
