# 📊 Архитектура Базы Данных Daralla

Проект переведен на **единую базу данных** для обеспечения целостности данных, упрощения поддержки и повышения производительности.

## 📍 Основной файл: `data/daralla.db`

Все данные хранятся в одном файле SQLite. Это позволяет делать атомарные бэкапы и избегать рассинхронизации между разными частями системы.

---

## 🛠 Таблицы и их назначение

### 1. `users` (Пользователи)
Центральная таблица для всех пользователей бота.
- `id`: Внутренний целочисленный ID (Primary Key).
- `user_id`: Telegram ID пользователя (String, Unique).
- `first_seen`: Timestamp первого запуска бота.
- `last_seen`: Timestamp последней активности.

### 2. `subscriptions` (Подписки)
Хранит информацию о купленных доступах.
- `id`: ID подписки.
- `subscriber_id`: Ссылка на `users.id`.
- `status`: `active` (активна), `expired` (истекла), `canceled` (отменена).
- `period`: `month` или `3month`.
- `expires_at`: Timestamp окончания действия.
- `subscription_token`: Уникальный токен для формирования VLESS-ссылки.
- `name`: Название подписки (может быть изменено пользователем).

### 3. `subscription_servers` (Серверы подписок)
Связывает подписку с конкретными VPN-серверами.
- `subscription_id`: Ссылка на `subscriptions.id`.
- `server_name`: Имя сервера из конфигурации.
- `client_email`: Email клиента в X-UI.
- `client_id`: UUID клиента (если применимо).

### 4. `payments` (Платежи)
История транзакций.
- `payment_id`: ID транзакции в YooKassa.
- `user_id`: Telegram ID.
- `status`: `pending`, `succeeded`, `canceled`, `expired`.
- `meta`: JSON с данными о периоде и типе подписки.
- `activated`: Флаг (0/1), была ли уже выдана подписка по этому платежу.

### 5. `sent_notifications` (Лог уведомлений)
Предотвращает повторную отправку одинаковых уведомлений.
- `user_id`, `subscription_id`, `notification_type`.
- `sent_at`: Timestamp отправки.

### 6. `config` (Конфигурация)
Динамические настройки бота.
- `key`: Название настройки.
- `value`: Значение.
- `description`: Описание для админ-панели.

## 🔄 Преимущества новой структуры

1.  **Отсутствие дублирования**: `user_id` теперь является единственным ключом, связывающим все таблицы.
2.  **Надежность**: Использование Foreign Keys (внешних ключей) гарантирует, что нельзя создать подписку для несуществующего пользователя.
3.  **Простой деплой**: Нужно переносить и проверять права только для одного файла `daralla.db`.
4.  **Единая инициализация**: Функция `init_all_db()` в `bot/db/__init__.py` создает все таблицы сразу при первом запуске.
