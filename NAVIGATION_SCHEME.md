# Полная схема навигации бота Daralla VPN

## Общая структура

```
/start (команда)
  └─> Главное меню
```

---

## 1. ГЛАВНОЕ МЕНЮ (main_menu)

**Состояние:** `NavStates.MAIN_MENU`  
**Callback:** `CallbackData.MAIN_MENU`  
**Обработчик:** `edit_main_menu`

### Кнопки:
- **"Купить"** → `CallbackData.BUY_VPN` → **Меню покупки**
- **"Мои подписки"** → `CallbackData.SUBSCRIPTIONS_MENU` → **Меню подписок**
- **"Инструкция"** → `CallbackData.INSTRUCTION` → **Меню инструкций**
- **"Наш канал"** → URL (внешняя ссылка)
- **"Админ-меню"** (только для админов) → `CallbackData.ADMIN_MENU` → **Админ меню**

### Переходы:
- `← Назад` → **Главное меню** (если есть предыдущее состояние)
- `main_menu` → **Главное меню** (очищает стек)

---

## 2. МЕНЮ ПОКУПКИ (buy_menu)

**Состояние:** `NavStates.BUY_MENU`  
**Callback:** `CallbackData.BUY_VPN` или `CallbackData.BUY_MENU`  
**Обработчик:** `buy_menu_handler`

### Кнопки:
- **"1 месяц — 150₽"** → `CallbackData.SELECT_PERIOD_MONTH` → **Обработка платежа**
- **"3 месяца — 350₽"** → `CallbackData.SELECT_PERIOD_3MONTH` → **Обработка платежа**
- **"← Назад"** → `CallbackData.BACK` → **Предыдущее меню** (обычно Главное меню)

### Переходы:
- После выбора периода → **Создание платежа** → **Ожидание оплаты** → **Успешная оплата** или **Ошибка оплаты**
- При успешной оплате → **Меню успешной оплаты** → кнопка "Мои подписки" → **Меню подписок**

---

## 3. МЕНЮ ПОДПИСОК (subs_menu)

**Состояние:** `NavStates.SUBSCRIPTIONS_MENU`  
**Callback:** `CallbackData.SUBSCRIPTIONS_MENU` или `CallbackData.MY_SUBSCRIPTIONS`  
**Обработчик:** `mykey`  
**Команда:** `/mykey` или `/mysubs`

### Кнопки (если есть подписки):
- **"[Название подписки]"** → `view_sub:{subscription_id}` → **Детали подписки**
- **"← Назад"** / **"Вперед →"** → `subs_page_{page}` → **Пагинация списка подписок**
- **"➕ Купить новую подписку"** → `CallbackData.BUY_MENU` → **Меню покупки**
- **"← Назад"** → `CallbackData.BACK` → **Предыдущее меню**

### Кнопки (если нет подписок):
- **"Купить подписку"** → `CallbackData.BUY_MENU` → **Меню покупки**
- **"← Назад"** → `CallbackData.BACK` → **Предыдущее меню**

---

## 4. ДЕТАЛИ ПОДПИСКИ (view_sub)

**Состояние:** `NavStates.SUBSCRIPTIONS_MENU` (остается в том же состоянии)  
**Callback:** `view_sub:{subscription_id}`  
**Обработчик:** `mykey` (с параметром subscription_id)

### Кнопки (если подписка активна):
- **"Продлить подписку"** → `extend_sub:{subscription_id}` → **Меню продления подписки**
- **"✏️ Переименовать"** → `rename_sub:{subscription_id}` → **Ввод нового имени** (текстовое сообщение)
- **"← Назад к списку"** → `CallbackData.SUBSCRIPTIONS_MENU` → **Меню подписок**

### Кнопки (если подписка истекла):
- **"Продлить подписку"** → `extend_sub:{subscription_id}` → **Меню продления подписки**
- **"← Назад к списку"** → `CallbackData.SUBSCRIPTIONS_MENU` → **Меню подписок**

---

## 5. МЕНЮ ПРОДЛЕНИЯ ПОДПИСКИ (extend_sub)

**Состояние:** `NavStates.SUBSCRIPTIONS_MENU` (остается в том же состоянии)  
**Callback:** `extend_sub:{subscription_id}` → затем `ext_sub_per:{period}:{subscription_id}`  
**Обработчик:** `extend_subscription_callback` → `extend_subscription_period_callback`

### Кнопки:
- **"1 месяц - 150₽"** → `ext_sub_per:month:{subscription_id}` → **Создание платежа на продление**
- **"3 месяца - 350₽"** → `ext_sub_per:3month:{subscription_id}` → **Создание платежа на продление**
- **"← Назад"** → `CallbackData.SUBSCRIPTIONS_MENU` → **Меню подписок**

### Переходы:
- После выбора периода → **Создание платежа** → **Ожидание оплаты** → **Успешное продление** → **Меню подписок**

---

## 6. ПЕРЕИМЕНОВАНИЕ ПОДПИСКИ (rename_sub)

**Состояние:** `NavStates.SUBSCRIPTIONS_MENU` (остается в том же состоянии)  
**Callback:** `rename_sub:{subscription_id}`  
**Обработчик:** `mykey` → затем `text_message_handler` (текстовое сообщение)

### Процесс:
1. Нажатие "✏️ Переименовать" → показывается запрос на ввод нового имени
2. Пользователь отправляет текстовое сообщение с новым именем
3. Подписка переименовывается → возврат в **Меню подписок**

### Кнопки:
- **"← Назад"** → `CallbackData.SUBSCRIPTIONS_MENU` → **Меню подписок**
- **"← Отмена"** → `CallbackData.SUBSCRIPTIONS_MENU` → **Меню подписок**

## 20. РАССЫЛКА (admin_broadcast)

**Состояние:** `NavStates.ADMIN_BROADCAST`  
**Callback:** `CallbackData.ADMIN_BROADCAST_START`  
**Обработчик:** `admin_broadcast_start`  
**Тип:** ConversationHandler (многошаговый процесс)

### Шаг 1: Начало рассылки
**Кнопки:**
- **"← Назад"** → `CallbackData.ADMIN_BROADCAST_BACK` → **Админ меню**

**Процесс:**
- Пользователь отправляет текстовое сообщение, фото, видео, документ, аудио, голосовое или стикер
- Сообщение сохраняется в `context.user_data['broadcast_text']`

### Шаг 3: Предпросмотр рассылки
**Кнопки:**
- **"Отправить"** → `CallbackData.ADMIN_BROADCAST_SEND` → **Отправка рассылки**
- **"← Назад"** → `CallbackData.ADMIN_BROADCAST_BACK` → **Отмена рассылки** → **Админ меню**

### Шаг 4: Результаты рассылки
**Кнопки:**
- **"Экспорт CSV"** → `CallbackData.ADMIN_BROADCAST_EXPORT` → **Экспорт результатов**

## 21. ОБРАБОТКА ПЛАТЕЖЕЙ

### Процесс покупки новой подписки:
1. **Меню покупки** → выбор периода → **Создание платежа** → **Ожидание оплаты**
2. При успешной оплате (webhook) → **Меню успешной оплаты** → кнопка "Мои подписки" → **Меню подписок**
3. При ошибке оплаты → **Меню ошибки оплаты** → кнопка "Попробовать снова" → **Меню покупки**

### Процесс продления подписки:
1. **Меню подписок** → "Продлить подписку" → **Меню продления** → выбор периода → **Создание платежа** → **Ожидание оплаты**
2. При успешной оплате (webhook) → **Меню успешного продления** → кнопка "Мои подписки" → **Меню подписок**
3. При ошибке оплаты → **Меню ошибки оплаты** → кнопка "Попробовать снова" → **Меню подписок**

## 25. НАВИГАЦИОННЫЙ СТЕК

Система использует навигационный стек для отслеживания истории переходов:

### Принцип работы:
- При переходе в новое меню состояние добавляется в стек
- При нажатии "← Назад" удаляется последнее состояние и возвращается предыдущее
- При нажатии "Главное меню" стек очищается и добавляется только главное меню

### Пример стека:
```
[main_menu] → [buy_menu] → [payment] → [subs_menu]
```

При нажатии "← Назад" из `subs_menu`:
```
[main_menu] → [buy_menu] → [payment]
```

## 27. ПОЛНЫЙ СПИСОК CALLBACK_DATA

### Основные навигационные:
- `back` - кнопка "Назад"
- `main_menu` - главное меню
- `instruction` - меню инструкций
- `buy_vpn` - меню покупки
- `my_subs` / `subs_menu` - меню подписок
- `admin_menu` - админ меню

### Инструкции:
- `instr_android` - инструкция Android
- `instr_ios` - инструкция iOS
- `instr_windows` - инструкция Windows
- `instr_macos` - инструкция macOS
- `instr_linux` - инструкция Linux
- `instr_tv` - инструкция Android TV
- `instr_faq` - FAQ

### Покупка:

## 29. ОСОБЕННОСТИ НАВИГАЦИИ

### Кнопка "Назад":
- Всегда ведет в предыдущее меню из навигационного стека
- Если стек пуст, ведет в главное меню

### Кнопка "Главное меню":
- Очищает навигационный стек
- Переходит в главное меню

### ConversationHandler:
- Рассылка использует ConversationHandler для многошагового процесса
- Состояния: `BROADCAST_WAITING_TEXT`, `BROADCAST_CONFIRM`

### Динамические callback_data:
- Многие callback_data содержат ID (например, `view_sub:{id}`)
- Обработчики используют паттерны для их обработки

### Обработка текстовых сообщений:
- Переименование подписки: ожидает текстовое сообщение после нажатия "Переименовать"
- Рассылка: ожидает текстовое сообщение/медиа после начала рассылки

/start

                → "1 месяц - 150₽"
                  → Создание платежа
                    → Ожидание оплаты
                      → Успешное продление (webhook)
                        → Меню подписок
```

### Пример 3: Админ - поиск пользователя и управление подпиской
```
/start
  → Главное меню

                → "← Назад"

## 31. КОМАНДЫ И ИХ НАВИГАЦИЯ

### `/start`
- Очищает навигационный стек
- Добавляет `main_menu` в стек
- Показывает главное меню

### `/mykey` или `/mysubs`
- Переходит в `subs_menu`
- Показывает список подписок

### `/instruction`
- Переходит в `instruction_menu`
- Показывает меню инструкций

### `/admin_user [user_id]`
- Если `user_id` указан: показывает информацию о пользователе
- Если `user_id` не указан: показывает форму поиска

## 32. ОБРАБОТКА ОШИБОК

### При ошибке оплаты:
- Показывается меню ошибки
- Кнопка "Попробовать снова" → возврат в меню покупки или меню подписок

### При ошибке создания подписки:
- Показывается сообщение об ошибке
- Кнопка "Попробовать снова" → возврат в меню покупки

### При ошибке синхронизации:
- Логируется ошибка
- Пользователь остается в текущем меню

---

## 33. WEBHOOK ОБРАБОТКА

### Успешная оплата новой подписки:
- Создается подписка в БД
- Создаются клиенты на всех серверах
- Показывается меню успешной оплаты
- Кнопка "Мои подписки" → меню подписок

### Успешное продление подписки:
- Обновляется `expires_at` в БД
- Синхронизируется время на всех серверах
- Показывается меню успешного продления
- Кнопка "Мои подписки" → меню подписок

## 34. ПАГИНАЦИЯ

### В меню подписок:
- Если подписок больше 5, показывается пагинация
- Кнопки "← Назад" и "Вперед →" для перехода между страницами
- Callback: `subs_page_{page}`

## 35. ЗАВИСИМОСТИ МЕНЮ

### Меню, доступные только админам:
- Все меню, начинающиеся с `admin_*`
- Кнопка "Админ-меню" в главном меню

### Меню, требующие активную подписку:

### Меню, требующие наличие подписок:

Каждое меню может иметь связанное изображение через `menu_type`:

## 37. ЗАМЕТКИ

1. **Навигационный стек** сохраняет историю переходов для кнопки "Назад"
2. **ConversationHandler** используется только для рассылки
3. **Динамические callback_data** обрабатываются через паттерны в `bot.py`
4. **Текстовые сообщения** обрабатываются только в контексте переименования или рассылки
5. **Webhook обработка** происходит асинхронно и не блокирует навигацию
6. **Админские функции** проверяют `ADMIN_IDS` перед выполнением
7. **Все кнопки "Назад"** используют единый обработчик через `NavigationIntegration`
