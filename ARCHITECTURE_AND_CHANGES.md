### Поток данных при покупке подписки

```
1. Пользователь → /start → Главное меню
2. Пользователь → "Купить VPN" → Выбор периода (1/3 месяца)
3. Пользователь → Выбор периода → Создание платежа (YooKassa)
4. YooKassa → Webhook → process_payment_webhook()
5. Webhook → process_successful_payment()
   ├─ Создание подписки в БД (SubscriptionManager.create_subscription_for_user)
   │  └─ device_limit = 1 (по умолчанию)
   ├─ Привязка всех серверов к подписке (attach_server_to_subscription)
   └─ Создание клиентов на серверах (ensure_client_on_server)
      ├─ Для каждого сервера:
      │  ├─ Проверка существования клиента
      │  ├─ Если нет → xui.addClient(limit_ip=device_limit)
      │  └─ Если есть → синхронизация expiry и limitIp
      └─ Обновление статуса платежа → 'completed'
6. Пользователь → Получение ключей → "Мои подписки"
```

### Поток синхронизации

```
Фоновая задача (каждые 6 часов):
1. SyncManager.run_sync()
   ├─ cleanup_expired_subscriptions (удаление >3 дней)
   ├─ sync_servers_with_config (добавление/удаление серверов)
   └─ sync_all_clients_states (синхронизация expiry и limitIp)
      └─ Для каждой активной подписки:
         └─ Для каждого сервера:
            └─ ensure_client_on_server()
               ├─ Проверка времени истечения (bidirectional sync)
               └─ Проверка limitIp (обновление если отличается)
```

### Поток уведомлений

```
Фоновая задача (каждые 5 минут):
1. NotificationManager._check_expiring_subscriptions()
   ├─ Получение всех активных подписок
   ├─ Для каждой подписки:
   │  ├─ Вычисление времени до истечения
   │  ├─ Если ≤ 3 дня → уведомление "3days"
   │  ├─ Если ≤ 1 день → уведомление "1day"
   │  └─ Если ≤ 1 час → уведомление "1hour"
   └─ Проверка дубликатов (не отправлять повторно за 24 часа)
```

## 🔄 Что изменилось

### Раньше (старая версия)

- ❌ **Один ключ с одного сервера**: При покупке пользователь получал один ключ с одного случайного сервера
- ❌ **Нет базы данных**: Все данные хранились в памяти или получались через циклы по серверам
- ❌ **Нет подписок**: Не было концепции подписки, только отдельные ключи
- ❌ **Нет синхронизации**: Время истечения и лимиты не синхронизировались между серверами
- ❌ **Нет уведомлений**: Пользователи не получали напоминания об истечении
- ❌ **Нет админ-панели**: Управление пользователями и ключами было затруднено

### Сейчас (новая версия)

- ✅ **Система подписок**: Каждая покупка создаёт подписку с уникальным токеном
- ✅ **Мультисерверность**: Одна подписка = ключи на всех доступных серверах
- ✅ **База данных SQLite**: Все данные хранятся в БД (пользователи, подписки, платежи, уведомления)
- ✅ **Автоматическая синхронизация**: 
  - Время истечения синхронизируется между БД и серверами (bidirectional)
  - Лимит IP (limitIp) синхронизируется из БД на все серверы
  - Автоматическое создание клиентов на новых серверах
- ✅ **Уведомления об истечении**: 
  - За 3 дня до истечения
  - За 1 день до истечения
  - За 1 час до истечения
- ✅ **Админ-панель**:
  - Поиск пользователей
  - Управление подписками (продление, отмена, изменение лимита IP)
  - Просмотр логов и метрик
  - Проверка состояния серверов
  - Ручная синхронизация
  - Рассылка сообщений
- ✅ **Навигационная система**: Единый стек навигации с кнопками "Назад" и "Главное меню"
- ✅ **Мониторинг серверов**: Автоматическая проверка здоровья серверов и уведомления админу
- ✅ **Метрики и аналитика**: Отслеживание эффективности уведомлений и статистика

### Ключевые улучшения

1. **Надёжность**: Данные хранятся в БД, не теряются при перезапуске
2. **Масштабируемость**: Легко добавлять новые серверы, они автоматически подключаются к существующим подпискам
3. **Удобство**: Пользователи могут иметь несколько подписок, управлять ими через бота
4. **Поддержка**: Админы могут быстро найти пользователя, продлить подписку, изменить лимиты
5. **Автоматизация**: Синхронизация и уведомления работают автоматически в фоне

## 📊 Технические детали

### Компоненты системы

- **bot.py**: Главный файл, инициализация бота и регистрация handlers
- **navigation/**: Система навигации (NavStates, NavigationManager, MenuHandlers)
- **services/**: Бизнес-логика (SubscriptionManager, SyncManager, NotificationManager, ServerManager)
- **handlers/**: Обработчики команд, callback'ов, админ-панели
- **db/**: Работа с базой данных (SQLite через aiosqlite)
- **utils/**: Вспомогательные функции (UI, message helpers)

### Фоновые задачи

1. **Синхронизация** (каждые 6 часов):
   - Очистка истёкших подписок (>3 дней)
   - Синхронизация списка серверов
   - Синхронизация времени истечения и limitIp

2. **Уведомления** (каждые 5 минут):
   - Проверка истекающих подписок
   - Отправка напоминаний

3. **Мониторинг серверов** (каждые 5 минут):
   - Проверка доступности серверов
   - Уведомления админу о проблемах

4. **Очистка платежей** (каждый час):
   - Удаление просроченных pending платежей

### Безопасность

- Проверка прав доступа для админ-функций
- Валидация входных данных
- Защита от спама уведомлений (24 часа между одинаковыми)
- Обработка ошибок и логирование
