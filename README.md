# Daralla VPN

Telegram-бот и веб-приложение для продажи и управления VPN-подписками. Интеграция с панелью **3x-ui** (X-UI), оплата через **YooKassa** (и опционально CryptoCloud).

## Возможности

- **Telegram Mini App** — управление подписками и оплата прямо в Telegram
- **Веб-клиент** — лендинг, регистрация/вход по логину и паролю, покупка подписки
- **Оплата** — YooKassa (карты), опционально CryptoCloud (крипто)
- **Панель 3x-ui** — автоматическое создание пользователей и подписок на серверах X-UI
- **Админ-панель** — серверы, подписки, пользователи, рассылка, статистика и графики
- **Мультисервер** — одна подписка может включать несколько серверов (по локациям)
- **Модуль событий** — реферальные конкурсы (опционально)

## Требования

- Python 3.11+
- База SQLite (через aiosqlite)
- Доступ к API панели 3x-ui на серверах VPN

## Установка

1. Клонируйте репозиторий и перейдите в каталог проекта.

2. Создайте виртуальное окружение и установите зависимости:

   ```bash
   python -m venv venv
   venv\Scripts\activate   # Windows
   pip install -r requirements.txt
   ```

3. Скопируйте конфиг и заполните переменные:

   ```bash
   copy .env.example .env
   ```

   В `.env` обязательно укажите:
   - `TELEGRAM_TOKEN` — токен бота от [@BotFather](https://t.me/BotFather)
   - `ADMIN_ID` или `ADMIN_IDS` — ID администратора(ов) через запятую
   - `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY` — данные магазина YooKassa
   - `WEBHOOK_URL` — HTTPS-URL для webhook YooKassa (например `https://ваш-домен/webhook/yookassa`)
   - `WEBAPP_URL` — URL мини-приложения (например `https://ваш-домен/`)

   Остальные переменные описаны в `.env.example` (бренд, цены, изображения, CryptoCloud, модуль событий и т.д.).

4. Подготовьте изображения для меню (по желанию):
   - `images/main_menu.jpg` — главное меню
   - `images/payment_success.jpg` — успешная оплата
   - `images/payment_failed.jpg` — ошибка оплаты  

   Пути можно переопределить в `.env`.

5. Запуск:

   ```bash
   python -m bot.bot
   ```

   Запускаются:
   - бот Telegram (polling);
   - веб-сервер Quart/Hypercorn на порту из `WEBHOOK_PORT` (по умолчанию 5000) для webhook и раздачи веб-приложения.

## Docker

```bash
docker build -t daralla .
docker run --env-file .env -p 5000:5000 daralla
```

Порт 5000 должен быть доступен снаружи для webhook и веб-приложения. За проксированием HTTPS (nginx/traefik и т.п.) следите отдельно.

| `scripts/` | Вспомогательные скрипты (миграции и т.д.) |

## Админ-панель

Доступна по маршрутам вида `/admin/...` (проверка по `ADMIN_ID`/`ADMIN_IDS`). В админке:

- управление серверами X-UI (добавление, настройка);
- просмотр и управление подписками и пользователями;
- рассылка в Telegram;
- статистика и графики;
- проверка здоровья сервисов.

Серверы добавляются через админ-панель; при старте бот подхватывает конфиг из БД.
