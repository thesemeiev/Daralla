# 📊 Визуальные диаграммы процессов

## 1. Полный цикл покупки подписки

```
┌─────────────┐
│  Пользователь │
│  нажимает     │
│  "Купить VPN" │
└──────┬───────┘
       │
       ▼
┌─────────────────────────┐
│  Бот показывает меню    │
│  выбора периода:        │
│  • 1 месяц - 150₽       │
│  • 3 месяца - 350₽      │
└──────┬──────────────────┘
       │
       ▼
┌─────────────────────────┐
│  Пользователь выбирает  │
│  период (например:      │
│  "1 месяц")             │
└──────┬──────────────────┘
       │
       ▼
┌─────────────────────────┐
│  handle_payment()       │
│  создает платеж в       │
│  YooKassa               │
└──────┬──────────────────┘
       │
       ▼
┌─────────────────────────┐
│  БД: daralla.db         │
│  Таблица: payments      │
│  status: 'pending'      │
└──────┬──────────────────┘
       │
       ▼
┌─────────────────────────┐
│  Бот отправляет         │
│  пользователю ссылку    │
│  на оплату              │
└──────┬──────────────────┘
       │
       ▼
┌─────────────────────────┐
│  Пользователь оплачивает│
│  через YooKassa         │
└──────┬──────────────────┘
       │
       ▼
┌─────────────────────────┐
│  YooKassa отправляет     │
│  webhook на              │
│  /webhook/yookassa       │
│  status: 'succeeded'     │
└──────┬───────────────────┘
       │
       ▼
┌─────────────────────────┐
│  process_payment_webhook()│
│  получает webhook        │
└──────┬──────────────────┘
       │
       ▼
┌─────────────────────────┐
│  process_new_purchase_  │
│  payment()              │
│  ├─ Создает подписку    │
│  ├─ Генерирует токен    │
│  ├─ Привязывает серверы │
│  └─ Создает клиентов    │
└──────┬──────────────────┘
       │
       ▼
┌─────────────────────────┐
│  БД: daralla.db         │
│  Таблица: subscriptions │
│  status: 'active'       │
│  token: xyz789          │
└──────┬──────────────────┘
       │
       ▼
┌─────────────────────────┐
│  Бот отправляет         │
│  пользователю ссылку:   │
│  /sub/xyz789            │
└─────────────────────────┘
```

## 3. Работа фоновых задач

```
┌─────────────────────────────────────────┐
│  Запуск бота (bot.py)                   │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│  on_startup()                            │
│  ├─ Инициализация daralla.db             │
│  ├─ Инициализация менеджеров             │
│  └─ Запуск фоновых задач                 │
└──────────────┬──────────────────────────┘
               │
               ├─────────────────────────────────┐
               │                                 │
               ▼                                 ▼
┌──────────────────────────┐    ┌──────────────────────────┐
│  cleanup_old_payments_   │    │  sync_db_with_xui_task()  │
│  task()                  │    │                           │
│  Каждый час:            │    │  Каждые 6 часов:         │
│  ├─ Удаляет старые      │    │  ├─ Проверяет подписки    │
│  │  pending платежи     │    │  ├─ Создает отсутствующих│
│  ├─ Удаляет старые      │    │  │  клиентов             │
│  │  записи              │    │  └─ Ищет orphaned        │
│  └─ Обновляет истекшие  │    └──────────────────────────┘
│     подписки            │
└──────────────────────────┘
               │
               ├─────────────────────────────────┐
               │                                 │
               ▼                                 ▼
┌──────────────────────────┐    ┌──────────────────────────┐
│  sync_servers_with_     │    │  NotificationManager      │
│  config_task()          │    │  Каждые 5 минут:         │
│  Каждый час:            │    │  ├─ Проверяет подписки   │
│  ├─ Добавляет новые     │    │  ├─ Определяет тип       │
│  │  серверы             │    │  │  уведомления         │
│  ├─ Удаляет старые      │    │  ├─ Проверяет БД (спам)  │
│  │  серверы             │    │  └─ Отправляет           │
│  └─ Создает клиентов    │    └──────────────────────────┘
│     на новых           │
└──────────────────────────┘
               │
               ▼
┌──────────────────────────┐
│  server_health_monitor() │
│  Каждые 5 минут:        │
│  ├─ Проверяет серверы   │
│  ├─ Отправляет уведомл. │
│  │  при изменении       │
│  └─ Логирует статус     │
└──────────────────────────┘
```

## 4. Структура базы данных (Единая)

```
┌─────────────────────────────────────────┐
│  daralla.db                             │
├─────────────────────────────────────────┤
│  users (Все пользователи)               │
│  ├─ id (PK)                             │
│  ├─ user_id (UNIQUE)                    │
│  ├─ first_seen                          │
│  └─ last_seen                           │
│                                          │
│  subscriptions (Подписки)               │
│  ├─ id (PK)                             │
│  ├─ subscriber_id (FK)                  │
│  ├─ status (active/expired)             │
│  ├─ period                              │
│  ├─ expires_at                          │
│  ├─ subscription_token (UNIQUE)         │
│  └─ name                                │
│                                          │
│  subscription_servers (Привязка к VPN)  │
│  ├─ id (PK)                             │
│  ├─ subscription_id (FK)                │
│  ├─ server_name                         │
│  └─ client_email                        │
│                                          │
│  payments (История оплат)               │
│  ├─ payment_id (PK)                     │
│  ├─ user_id                             │
│  ├─ status (pending/succeeded/...)      │
│  └─ meta (JSON)                         │
│                                          │
│  sent_notifications (Лог спама)         │
│  ├─ user_id                             │
│  ├─ subscription_id                     │
│  └─ notification_type                   │
└─────────────────────────────────────────┘
```

## ✅ Выводы

Переход на единую БД сделал систему:
- **Надежной**: Меньше риск повреждения файлов.
- **Простой**: Легче делать бэкапы.
- **Быстрой**: Все данные в одном месте.
