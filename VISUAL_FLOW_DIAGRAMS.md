## 6. Процесс синхронизации

```
┌─────────────────────────┐
│  sync_db_with_xui_task()│
│  (каждые 6 часов)        │
└──────────┬───────────────┘
           │
           ▼
┌─────────────────────────┐
│  Для каждой активной    │
│  подписки:              │
└──────────┬───────────────┘
           │
           ▼
┌─────────────────────────┐
│  Получаем серверы       │
│  подписки из БД         │
└──────────┬───────────────┘
           │
           ▼
┌─────────────────────────┐
│  Для каждого сервера:   │
│  ├─ Проверяем: есть ли  │
│  │  клиент на сервере?  │
│  ├─ Если нет → создаем   │
│  └─ Если есть →          │
│     синхронизируем       │
└──────────┬───────────────┘
           │
           ▼
┌─────────────────────────┐
│  Ищем orphaned клиентов │
│  (есть на сервере, но   │
│  нет в БД)              │
└─────────────────────────┘
```

```
┌─────────────────────────┐
│  NotificationManager    │
│  (каждые 5 минут)       │
└──────────┬───────────────┘
           │
           ▼
┌─────────────────────────┐
│  Получаем все активные  │
│  подписки               │
└──────────┬───────────────┘
           │
           ▼
┌─────────────────────────┐
│  Для каждой подписки:   │
│  ├─ Вычисляем: сколько  │
│  │  дней до истечения?  │
│  ├─ Определяем тип:     │
│  │  • 3 дня → 3days     │
│  │  • 1 день → 1day     │
│  │  • 1 час → 1hour     │
└──────────┬───────────────┘
           │
           ▼
┌─────────────────────────┐
│  Проверяем БД:         │
│  отправляли ли уже      │
│  это уведомление?      │
└──────────┬───────────────┘
           │
           ├─── Да ────► Пропускаем
           │
           ▼
           Нет
           │
           ▼
┌─────────────────────────┐
│  Отправляем уведомление   │
│  пользователю             │
└──────────┬────────────────┘
           │
           ▼
┌─────────────────────────┐
│  Записываем в БД:       │
│  уведомление отправлено  │
└─────────────────────────┘
```

## 🎯 Ключевые моменты

1. **Все процессы асинхронные** - используют `async/await`
2. **БД - источник истины** - все данные хранятся в БД
3. **Фоновые задачи работают постоянно** - автоматически
4. **Обработка ошибок** - все функции обрабатывают исключения
5. **Логирование** - все действия логируются
