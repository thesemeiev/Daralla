# 📖 Руководство по архитектуре Daralla (Единая БД)

Это руководство поможет вам понять, как устроен проект после перехода на единую базу данных.

### 🏗 Общая архитектура

Бот построен на модульной структуре:
- **`bot/db/`**: Работа с базой данных (теперь всё в одном файле `daralla.db`).
- **`bot/handlers/`**: Обработка команд, нажатий кнопок и вебхуков платежей.
- **`bot/services/`**: Бизнес-логика (управление подписками, уведомления, синхронизация с серверами).
- **`bot/core/`**: Ядро бота (запуск, фоновые задачи).
- **`bot/navigation/`**: Централизованная система меню.

---

### 🗄 База данных: `daralla.db`

Теперь у бота **всего одна база данных** вместо четырех. Это упрощает бэкапы и ускоряет работу.

Внутри `daralla.db` находятся следующие таблицы:

1.  **`users`**: Все пользователи бота.
    - `user_id`: Telegram ID.
    - `first_seen`: Когда впервые зашел.
    - `last_seen`: Последняя активность.

2.  **`subscriptions`**: Купленные подписки.
    - `subscriber_id`: Ссылка на пользователя.
    - `status`: Состояние (`active`, `expired`).
    - `expires_at`: Дата окончания.
    - `subscription_token`: Уникальный ключ для ссылки.

3.  **`subscription_servers`**: Связь подписки с X-UI серверами.
    - Хранит информацию о том, на каких серверах создан клиент для этой подписки.

4.  **`payments`**: История оплат.
    - Все попытки оплаты через YooKassa и их статус.

5.  **`sent_notifications`**: Лог уведомлений.
    - Чтобы бот не спамил и знал, когда он уже напоминал об оплате.

6.  **`config`**: Настройки бота.
    - Цены, лимиты и другие параметры, которые можно менять через админку.

---

### 💳 Процесс оплаты (Workflow)

1.  **Заказ**: Пользователь выбирает период в меню "Купить".
2.  **Платеж**: Бот создает платеж в YooKassa и записывает его в таблицу `payments` со статусом `pending`.
3.  **Вебхук**: Когда пользователь оплачивает, YooKassa присылает сигнал (вебхук).
4.  **Активация**: 
    - Бот меняет статус в `payments` на `succeeded`.
    - Создает запись в `subscriptions`.
    - `MultiServerManager` создает клиентов на всех доступных VPN-серверах.
5.  **Уведомление**: Бот поздравляет пользователя и присылает ссылку на подписку.

---

### 🕒 Фоновые задачи (Background Tasks)

Бот постоянно выполняет несколько задач:
- **`check_expiring_subscriptions`**: Раз в час проверяет, у кого скоро кончится подписка, и шлет уведомления.
- **`sync_all_subscriptions`**: Раз в 6 часов проверяет, что все клиенты созданы на всех серверах (если сервер был в офлайне, клиент добавится позже).
- **`cleanup_expired_payments`**: Раз в час удаляет старые неоплаченные счета.

---

### 🚀 Деплой и Бэкапы

- **Деплой**: Происходит автоматически через GitHub Actions (файлы в `.github/workflows/`).
- **Бэкапы**: Скрипт `backup_data.sh` каждый день упаковывает `daralla.db` в архив.
- **Права доступа**: Если база данных не открывается, используйте `./fix-database-permissions.sh`.
