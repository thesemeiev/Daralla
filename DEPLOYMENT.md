# 🚀 CI/CD Деплой Daralla Bot

## 📋 Обзор

Настроен полный CI/CD pipeline для автоматического деплоя бота на сервер при каждом push в main ветку.

## 🔧 Настройка GitHub Secrets

В настройках репозитория (Settings → Secrets and variables → Actions) добавьте:

### Обязательные секреты:
```
SERVER_HOST          # IP адрес или домен сервера
SERVER_USER          # Пользователь для SSH (обычно root или ubuntu)
SERVER_PASSWORD      # Пароль от сервера (или SERVER_SSH_KEY для ключа)
```

### Опциональные секреты:
```
SERVER_PORT          # SSH порт (по умолчанию 22)
```

## 🏗️ Структура CI/CD

### 1. **Тестирование** (`test` job)
- ✅ Проверка кода на ошибки
- ✅ Установка зависимостей
- ✅ Запуск тестов
- ✅ Линтинг кода

### 2. **Сборка** (`build` job)
- ✅ Сборка Docker образа
- ✅ Публикация в GitHub Container Registry
- ✅ Кэширование слоев

### 3. **Деплой** (`deploy` job)
- ✅ Автоматический деплой на сервер
- ✅ Создание резервной копии
- ✅ Обновление кода
- ✅ Перезапуск контейнера
- ✅ Проверка статуса

### 4. **Откат** (`rollback` job)
- ✅ Автоматический откат при ошибке
- ✅ Восстановление из бэкапа

## 📅 Автоматические задачи

### Ежедневные бэкапы
- 🕒 **Время:** 3:00 UTC (6:00 МСК)
- 💾 **Действие:** Создание резервной копии
- 🗑️ **Очистка:** Удаление бэкапов старше 7 дней

## 🛠️ Команды для управления

### На сервере:
```bash
# Ручной деплой
./deploy-server.sh

# Создание бэкапа
./backup.sh

# Откат к предыдущей версии
./rollback.sh

# Просмотр логов
docker-compose logs -f

# Статус контейнера
docker-compose ps
```

### В GitHub:
- **Ручной запуск деплоя:** Actions → Deploy Daralla Bot → Run workflow
- **Ручной бэкап:** Actions → Daily Backup → Run workflow

## 🔄 Workflow файлы

### `.github/workflows/deploy.yml`
Основной workflow для деплоя:
- Тестирование кода
- Сборка Docker образа
- Деплой на сервер
- Откат при ошибке

### `.github/workflows/backup.yml`
Ежедневные резервные копии:
- Автоматическое создание бэкапов
- Очистка старых файлов

## 🚨 Мониторинг

### Уведомления о статусе:
- ✅ Успешный деплой
- ❌ Ошибка деплоя
- 🔄 Автоматический откат

### Логи:
- GitHub Actions logs
- Docker container logs
- Application logs

## 🔐 Безопасность

- SSH ключи хранятся в GitHub Secrets
- Переменные окружения не попадают в логи
- Автоматические бэкапы перед деплоем
- Возможность быстрого отката

## 📊 Статус деплоя

Проверить статус можно:
1. В GitHub Actions
2. На сервере: `docker-compose ps`
3. В логах: `docker-compose logs -f`

## 🆘 Troubleshooting

### Проблема: Деплой не запускается
**Решение:** Проверьте GitHub Secrets

### Проблема: Ошибка SSH подключения
**Решение:** Проверьте SERVER_HOST, SERVER_USER, SERVER_SSH_KEY

### Проблема: Контейнер не запускается
**Решение:** Проверьте .env файл и логи

### Проблема: Нужен откат
**Решение:** Запустите `./rollback.sh` на сервере
