# Автоматическая синхронизация серверов с подписками

## Описание

Система автоматически синхронизирует серверы в подписках с текущей конфигурацией серверов (`SERVERS_BY_LOCATION` в `bot/bot.py`).

## Как это работает

### При добавлении нового сервера в конфигурацию:

1. **Автоматическое добавление в подписки:**
   - При следующей синхронизации (каждый час или при старте бота)
   - Система проверяет все активные подписки
   - Для каждой подписки находит серверы, которых нет в подписке, но есть в конфигурации
   - Автоматически создает клиента на новом сервере (с тем же email, что и на других серверах)
   - Привязывает новый сервер к подписке в БД

2. **Результат:**
   - Новый сервер автоматически появляется в подписке пользователя
   - Пользователь получает доступ к новому серверу без дополнительных действий
   - VLESS ссылки для нового сервера автоматически добавляются в subscription URL

### При удалении сервера из конфигурации:

1. **Автоматическое удаление из подписок:**
   - При следующей синхронизации (каждый час или при старте бота)
   - Система проверяет все активные подписки
   - Для каждой подписки находит серверы, которые есть в подписке, но больше нет в конфигурации
   - Удаляет связь сервера с подпиской в БД

2. **Результат:**
   - Удаленный сервер исчезает из подписки пользователя
   - VLESS ссылки для удаленного сервера больше не возвращаются в subscription URL
   - **Важно:** Клиент на сервере X-UI остается (не удаляется), но не используется в подписке

## Периодичность синхронизации

- **При старте бота:** Первоначальная синхронизация через 30 секунд после запуска
- **Периодически:** Каждый час автоматическая синхронизация
- **При ошибке:** Повторная попытка через 30 минут

## Логирование

Все действия синхронизации логируются:

```
INFO: Начало синхронизации серверов в подписках с конфигурацией
INFO: Найдено 5 активных подписок
INFO: В конфигурации 2 серверов: {'Latvia-1', 'Netherlands-1'}
INFO: Подписка 1 (user=6735703554): добавить 1 серверов, удалить 0 серверов
INFO: Клиент 6735703554_abc123 создан на сервере Netherlands-1 для подписки 1
INFO: Сервер Netherlands-1 добавлен в подписку 1
INFO: Синхронизация завершена: проверено 5 подписок, добавлено 1 серверов, удалено 0 серверов, создано 1 клиентов
```

## Пример использования

### Добавление нового сервера:

1. Добавьте сервер в `SERVERS_BY_LOCATION` в `bot/bot.py`:
   ```python
   SERVERS_BY_LOCATION = {
       "Latvia": [
           {
               "name": "Latvia-1",
               "display_name": "🇱🇻 🌐 Latvia Premium - 1",
               "host": os.getenv("XUI_HOST_LATVIA_1"),
               "login": os.getenv("XUI_LOGIN_LATVIA_1"),
               "password": os.getenv("XUI_PASSWORD_LATVIA_1"),
               "vpn_host": os.getenv("XUI_VPN_HOST_LATVIA_1")
           },
       ],
       "Netherlands": [
           {
               "name": "Netherlands-1",
               "display_name": "🇳🇱 🌐 Netherlands Premium - 1",
               "host": os.getenv("XUI_HOST_NETHERLANDS_1"),
               "login": os.getenv("XUI_LOGIN_NETHERLANDS_1"),
               "password": os.getenv("XUI_PASSWORD_NETHERLANDS_1"),
               "vpn_host": os.getenv("XUI_VPN_HOST_NETHERLANDS_1")
           },
       ],
       "USA": [  # Новый сервер
           {
               "name": "USA-1",
               "display_name": "🇺🇸 🌐 USA Premium - 1",
               "host": os.getenv("XUI_HOST_USA_1"),
               "login": os.getenv("XUI_LOGIN_USA_1"),
               "password": os.getenv("XUI_PASSWORD_USA_1"),
               "vpn_host": os.getenv("XUI_VPN_HOST_USA_1")
           },
       ],
   }
   ```

2. Добавьте переменные окружения в `.env`:
   ```env
   XUI_HOST_USA_1=http://192.145.32.100:44764
   XUI_LOGIN_USA_1=thesemeiev
   XUI_PASSWORD_USA_1=Abdu-Rahman_2506
   XUI_VPN_HOST_USA_1=192.145.32.100
   ```

3. Перезапустите бота:
   ```bash
   sudo docker compose down
   sudo docker compose up -d
   ```

4. **Автоматически:**
   - Через 30 секунд после старта выполнится первоначальная синхронизация
   - Новый сервер `USA-1` будет добавлен во все активные подписки
   - Для каждой подписки будет создан клиент на новом сервере
   - Пользователи автоматически получат доступ к новому серверу

### Удаление сервера:

1. Удалите сервер из `SERVERS_BY_LOCATION` в `bot/bot.py`

2. Перезапустите бота:
   ```bash
   sudo docker compose down
   sudo docker compose up -d
   ```

3. **Автоматически:**
   - Через 30 секунд после старта выполнится первоначальная синхронизация
   - Удаленный сервер будет удален из всех активных подписок
   - VLESS ссылки для удаленного сервера больше не будут возвращаться

## Ручная синхронизация

Если нужно выполнить синхронизацию вручную (например, через админ-команду), можно использовать:

```python
from bot.services.subscription_manager import SubscriptionManager

# Получить subscription_manager из глобальных переменных
subscription_manager = get_globals()['subscription_manager']

# Выполнить синхронизацию
stats = await subscription_manager.sync_servers_with_config(auto_create_clients=True)
```

## Важные замечания

1. **Email клиента:** Все серверы в одной подписке используют один и тот же email клиента. При добавлении нового сервера используется email из первого существующего сервера подписки.

2. **Срок действия клиента:**
   - **Основной источник истины:** Время истечения из БД подписки (`expires_at`)
   - **Синхронизация с существующими серверами:** Система проверяет время истечения клиентов на других серверах подписки
   - **Используется максимальное время:** Если на каком-то сервере время больше, чем в БД (например, было продлено вручную), используется максимальное значение
   - **Гарантия синхронизации:** Это гарантирует, что все серверы в подписке имеют одинаковое или максимальное время истечения
   
   **Пример:**
   - Время в БД: 16.01.2026 00:43:04
   - Время на сервере Latvia-1: 16.01.2026 00:43:04
   - Время на сервере Netherlands-1: 20.01.2026 12:00:00 (было продлено вручную)
   - **Результат:** Новый сервер получит время до 20.01.2026 12:00:00 (максимальное)

3. **Клиенты на удаленных серверах:** При удалении сервера из конфигурации клиент на X-UI сервере **не удаляется** (остается на сервере, но не используется в подписке). Это сделано для безопасности - чтобы случайно не удалить важные данные.

4. **Ошибки:** Если создание клиента на новом сервере не удалось, сервер все равно будет привязан к подписке в БД. Клиент можно создать вручную позже, и он автоматически будет использоваться.

5. **Производительность:** Синхронизация выполняется асинхронно и не блокирует работу бота. При большом количестве подписок процесс может занять некоторое время.
