# Как работает обновление конфигурации

## Ответ: **НОВЫЙ конфиг из X-UI** ✅

Когда VPN клиент запрашивает subscription через `/sub/<token>`, происходит следующее:

### 1. Запрос subscription endpoint
```
VPN клиент → GET /sub/<token>
```

### 2. Генерация VLESS ссылок
```python
subscription_manager.build_vless_links_for_subscription()
  ↓
xui.link(client_email)  # Для каждого сервера
  ↓
self.list()  # Запрос к X-UI API в реальном времени!
  ↓
GET /panel/api/inbounds/list
```

### 3. Парсинг и генерация ссылки
- Бот получает **актуальные данные** из X-UI API
- Парсит `streamSettings`, `realitySettings`, `xhttpSettings`
- Генерирует VLESS ссылку на основе **текущей конфигурации**

## Что хранится в БД:

✅ **Хранится:**
- Информация о подписке (статус, срок действия, токен)
- Связь подписка ↔ серверы (какие серверы привязаны, `client_email`)

❌ **НЕ хранится:**
- Настройки клиента (порт, протокол, Reality настройки)
- Параметры streamSettings
- Эти данные берутся из X-UI API каждый раз

## Вывод:

Если вы измените конфиг в X-UI и нажмете "обновить" в VPN клиенте:
- ✅ Подтянется **НОВЫЙ конфиг** из X-UI
- ✅ Изменения применятся сразу
- ❌ Старый конфиг из БД НЕ используется

## Исключение:

Если X-UI сервер недоступен в момент запроса, бот вернет ошибку или пустой список ссылок.
