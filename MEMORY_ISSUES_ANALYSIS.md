# Анализ потенциальных проблем с памятью

## 🔴 Критические проблемы

### 1. `extension_messages` - глобальный словарь без очистки
**Файл:** `bot/bot.py:267`
```python
extension_messages = {}  # Ключ: payment_id, Значение: (chat_id, message_id)
```

**Проблема:** Словарь накапливается бесконечно. Каждое продление подписки добавляет запись, но никогда не удаляется.

**Риск:** При большом количестве продлений (тысячи) может занять значительную память.

**Решение:** 
- Добавить TTL (время жизни) для записей
- Очищать старые записи периодически
- Удалять после успешной обработки

---

### 2. `get_all_user_ids()` - загрузка всех пользователей в память
**Файл:** `bot/db/users_db.py:24`

**Проблема:** При большом количестве пользователей (десятки тысяч) загружает всех в память сразу.

**Использование:**
- `admin_broadcast.py:178` - для рассылки всем пользователям

**Риск:** При 50,000+ пользователях может занять 10-50 МБ памяти.

**Решение:**
- Использовать пагинацию/батчинг
- Или использовать генератор вместо списка

---

### 3. `get_all_active_subscriptions()` - загрузка всех подписок
**Файл:** `bot/db/subscribers_db.py:89`

**Проблема:** Загружает все активные подписки в память сразу.

**Использование:**
- `sync_manager.py:138` - при синхронизации
- `notification_manager.py` - при проверке уведомлений

**Риск:** При 10,000+ подписках может занять 5-20 МБ памяти.

**Решение:**
- Обрабатывать батчами
- Использовать генератор

---

## 🟡 Средние проблемы

### 4. `broadcast_details` - ограничен, но может быть большим
**Файл:** `bot/handlers/admin/admin_broadcast.py:185, 216-222`

**Проблема:** Ограничен до 10,000 записей, но все равно может быть большим.

**Риск:** При рассылке 10,000 пользователям занимает ~1-2 МБ памяти.

**Решение:** 
- Уже ограничен до 10,000 (хорошо)
- Можно уменьшить до 1,000 или сохранять в БД

---

### 5. `context.user_data` - накапливается
**Проблема:** Telegram Bot API хранит `user_data` для каждого пользователя в памяти.

**Риск:** При большом количестве активных пользователей может накапливаться.

**Решение:**
- Telegram Bot API управляет этим автоматически
- Но можно периодически очищать неиспользуемые данные

---
