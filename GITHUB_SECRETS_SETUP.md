# 🔐 Настройка GitHub Secrets для деплоя

## 🎯 **Два способа аутентификации:**

### **Способ 1: По паролю (проще)**
### **Способ 2: По SSH ключу (безопаснее)**

---

## 🔑 **Способ 1: Аутентификация по паролю**

### **Настройка GitHub Secrets:**
1. Перейдите в ваш репозиторий на GitHub
2. **Settings** → **Secrets and variables** → **Actions**
3. Нажмите **"New repository secret"**
4. Добавьте эти секреты:

```
SERVER_HOST     = 192.168.1.100          # IP вашего сервера
SERVER_USER     = root                   # или ubuntu, или ваш_пользователь
SERVER_PASSWORD = your_server_password   # пароль от сервера
SERVER_PORT     = 22                     # опционально, по умолчанию 22
```

### **Преимущества:**
- ✅ **Простота настройки** - не нужно генерировать ключи
- ✅ **Быстро** - сразу готово к использованию
- ✅ **Понятно** - обычный пароль

### **Недостатки:**
- ❌ **Менее безопасно** - пароль в открытом виде
- ❌ **Может быть заблокирован** при множественных попытках входа

---

## 🔐 **Способ 2: Аутентификация по SSH ключу**

### **Генерация SSH ключа:**
```bash
# На вашем компьютере
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
# Нажмите Enter для всех вопросов (используйте стандартные пути)
```

### **Копирование ключа на сервер:**
```bash
# Скопируйте публичный ключ на сервер
ssh-copy-id your_user@your_server_ip

# Или вручную:
cat ~/.ssh/id_rsa.pub | ssh your_user@your_server_ip "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

### **Настройка GitHub Secrets:**
```
SERVER_HOST     = 192.168.1.100          # IP вашего сервера
SERVER_USER     = root                   # или ubuntu, или ваш_пользователь
SERVER_SSH_KEY  = -----BEGIN OPENSSH PRIVATE KEY-----  # содержимое ~/.ssh/id_rsa
SERVER_PORT     = 22                     # опционально, по умолчанию 22
```

### **Преимущества:**
- ✅ **Более безопасно** - ключ не передается по сети
- ✅ **Надежно** - не блокируется при множественных попытках
- ✅ **Стандартный способ** для серверов

### **Недостатки:**
- ❌ **Сложнее настройка** - нужно генерировать ключи
- ❌ **Больше шагов** - копирование ключей

---

## 🚀 **Рекомендации:**

### **Для тестирования:**
- Используйте **пароль** - быстрее настроить

### **Для продакшена:**
- Используйте **SSH ключ** - безопаснее

### **Если у вас уже есть SSH доступ:**
- Используйте **SSH ключ** - не нужно менять настройки

---

## 🔧 **Проверка настройки:**

### **Тест подключения по паролю:**
```bash
ssh your_user@your_server_ip
# Введите пароль
```

### **Тест подключения по ключу:**
```bash
ssh your_user@your_server_ip
# Должно подключиться без пароля
```

---

## ⚠️ **Важные моменты:**

1. **Пароль должен быть надежным** - минимум 12 символов
2. **SSH ключ должен быть в формате OpenSSH** (id_rsa, не id_ed25519)
3. **Проверьте права доступа** - пользователь должен иметь доступ к папке с ботом
4. **Убедитесь, что SSH включен** на сервере

---

## 🆘 **Troubleshooting:**

### **Ошибка: "Permission denied"**
- Проверьте правильность пароля/ключа
- Убедитесь, что пользователь существует на сервере

### **Ошибка: "Connection refused"**
- Проверьте IP адрес сервера
- Убедитесь, что SSH порт открыт (обычно 22)

### **Ошибка: "Host key verification failed"**
- Добавьте сервер в known_hosts: `ssh-keyscan your_server_ip >> ~/.ssh/known_hosts`
