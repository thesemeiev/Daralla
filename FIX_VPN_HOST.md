# Исправление проблемы с IP адресом в VLESS ссылках

## Проблема
VLESS ссылки генерируются с неправильным IP адресом (IP панели X-UI вместо IP VPN сервера), из-за чего нет пинга и подключение не работает.

## Решение

Добавлена поддержка отдельного IP/домена для VPN сервера. Теперь можно указать IP VPN сервера, который будет использоваться в VLESS ссылках.

### 1. Добавьте переменную в .env файл

```bash
cd /root/Daralla
sudo nano .env
```

Добавьте для каждого сервера переменную `XUI_VPN_HOST_<SERVER_NAME>`:

```env
# X-UI Server Configuration
XUI_HOST_LATVIA_1=http://185.113.139.11:8172
XUI_LOGIN_LATVIA_1=ваш_логин
XUI_PASSWORD_LATVIA_1=ваш_пароль

# VPN Server IP/Domain (для VLESS ссылок)
# Если VPN сервер на другом IP, укажите его здесь
XUI_VPN_HOST_LATVIA_1=192.145.28.122
```

**Важно:**
- `XUI_HOST_LATVIA_1` - это адрес панели управления X-UI (для API)
- `XUI_VPN_HOST_LATVIA_1` - это IP/домен VPN сервера (для VLESS ссылок)
- Если VPN сервер на том же IP, что и панель, можно не указывать `XUI_VPN_HOST_LATVIA_1`

### 2. Если используете домен вместо IP

```env
XUI_VPN_HOST_LATVIA_1=vpn.example.com
```

### 3. Перезапустите бота

```bash
cd /root/Daralla
sudo docker compose down
sudo docker compose up -d
```

### 4. Проверьте логи

```bash
sudo docker compose logs -f telegram-bot | grep "VPN host\|Используется"
```

Должны появиться сообщения:
```
Используется VPN host из конфигурации: 192.145.28.122
```

### 5. Протестируйте новую подписку

1. Создайте новый платеж
2. Получите ссылку на подписку
3. Проверьте, что в VLESS ссылке правильный IP

## Как узнать правильный IP VPN сервера?

### Вариант 1: Из конфига X-UI

1. Зайдите в панель X-UI
2. Посмотрите настройки inbound
3. Там должен быть указан IP сервера

### Вариант 2: Из текущего конфига

В конфиге, который вы показали, IP: `192.145.28.122`

### Вариант 3: Проверьте сетевые интерфейсы

```bash
# На VPN сервере
ip addr show
# или
ifconfig
```

Найдите внешний IP адрес сервера.

## Если VPN сервер на том же IP, что и панель

Если VPN сервер находится на том же IP, что и панель управления (`185.113.139.11`), то переменную `XUI_VPN_HOST_LATVIA_1` можно не указывать - будет использоваться IP панели.

## Проверка после настройки

1. ✅ Переменная `XUI_VPN_HOST_LATVIA_1` добавлена в `.env`
2. ✅ Бот перезапущен
3. ✅ В логах видно "Используется VPN host из конфигурации"
4. ✅ Новая VLESS ссылка содержит правильный IP
5. ✅ Пинг работает, подключение устанавливается
