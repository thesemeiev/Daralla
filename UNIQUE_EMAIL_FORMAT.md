# Формат уникального имени (email) для клиентов

## Текущий формат:

### 1. Для платной подписки (`payment_handler.py`):
```python
key_id = str(uuid.uuid4())  # Генерируется UUID4
unique_email = f'{user_id}_{key_id}'
```

**Формат:** `{telegram_user_id}_{uuid4}`

**Пример:** `6735703554_500074fa-62cb-4a44-961d-c8bd7aa09685`

### 2. Для бесплатного ключа (`start_handler.py`):
```python
unique_email = f"{user_id_str}_{uuid.uuid4()}"
```

**Формат:** `{telegram_user_id}_{uuid4}`

**Пример:** `6735703554_ed82258d-eb0b-4223-8788-f8117f47eaf2`

## Структура:

- **Первая часть:** Telegram User ID (например, `6735703554`)
- **Разделитель:** `_` (подчеркивание)
- **Вторая часть:** UUID4 (например, `500074fa-62cb-4a44-961d-c8bd7aa09685`)

## Где используется:

1. **В X-UI:** как `email` клиента
2. **В базе данных:** как `client_email` в таблице `subscription_servers`
3. **Для идентификации:** уникальный идентификатор клиента на каждом сервере

## Уникальность:

- ✅ Гарантируется UUID4 (вероятность коллизии практически нулевая)
- ✅ Один и тот же `unique_email` используется на всех серверах для одной подписки
- ✅ Разные подписки имеют разные `unique_email` (даже у одного пользователя)

## Примеры:

```
6735703554_500074fa-62cb-4a44-961d-c8bd7aa09685  # Подписка 1
6735703554_ed82258d-eb0b-4223-8788-f8117f47eaf2  # Подписка 2 (другая)
1234567890_abc12345-def6-7890-abcd-ef1234567890  # Другой пользователь
```
