# Объяснение работы device_limit и limitIp

## 📋 Текущая логика

### 1. При создании подписки (покупка):
- `device_limit` берется из `meta.get('device_limit', 1)` - **по умолчанию всегда 1**
- Сохраняется в БД в таблице `subscriptions`
- При создании клиента на сервере используется `device_limit` из БД

### 2. При создании клиента на сервере:
- Используется `device_limit` из подписки (из БД)
- Передается в `xui.addClient(..., limit_ip=device_limit)`
- Устанавливается `limitIp` на сервере

### 3. При синхронизации существующих клиентов:
- `sync_all_clients_states` получает `device_limit` из БД для каждой подписки
- Вызывает `ensure_client_on_server` с `device_limit` из БД
- `ensure_client_on_server` проверяет текущий `limitIp` на сервере
- Если `limitIp` отсутствует (None), равен 0, или отличается от `device_limit` - обновляет

### 4. При изменении device_limit через админ-панель:
- Обновляется в БД через `update_subscription_device_limit`
- Синхронизируется на всех серверах через `ensure_client_on_server`

## ✅ Что работает правильно:

1. **По умолчанию при покупке всегда 1** - `meta.get('device_limit', 1)`
2. **Можно менять в БД** - функция `admin_change_device_limit` уже реализована
3. **Синхронизация из БД на серверы** - `ensure_client_on_server` синхронизирует `limitIp`

## ⚠️ Проблема с существующими клиентами:

Если у существующих клиентов `limitIp` не установлен, нужно:
1. Запустить синхронизацию через `/admin_sync`
2. Или дождаться автоматической синхронизации
